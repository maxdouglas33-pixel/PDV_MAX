# Versionamento

Este projeto seguirá o padrão SemVer:

- `MAJOR.MINOR.PATCH`

## Regras

- `MAJOR`: mudanças incompatíveis ou reorganização grande do fluxo;
- `MINOR`: novas funcionalidades sem quebrar o uso atual;
- `PATCH`: correções, ajustes e melhorias pequenas.

## Processo sugerido

1. Alterar arquivos e validar localmente.
2. Atualizar `CHANGELOG.md`.
3. Incrementar `VERSION`.
4. Criar tag Git, por exemplo `v0.1.1`.

## Exemplo

- `0.1.0` - primeira base publicada;
- `0.1.1` - correção ou ajuste pequeno;
- `0.2.0` - nova melhoria relevante;
- `1.0.0` - versão estável principal.
